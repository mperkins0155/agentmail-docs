## Overview

Modern development tools enable you to build, test, and iterate on AgentMail applications directly in your browser with instant feedback. This guide covers interactive development environments, live preview tools, and AI-powered coding assistants.

## WebContainer-Based Development

WebContainers allow you to run Node.js directly in the browser, providing instant feedback and live previews.

### StackBlitz

StackBlitz uses WebContainers to provide a full development environment in your browser.

<Steps>
  <Step title="Quick Start">
    Click the button below to open a pre-configured AgentMail starter project:
    
    <Card
      title="Open in StackBlitz"
      icon="fa-solid fa-bolt"
      href="https://stackblitz.com/fork/node-agentmail-starter"
      target="_blank"
    />
    
    The starter includes:
    - AgentMail SDK pre-installed
    - Sample webhook handler
    - Live preview environment
    - Hot module reloading
  </Step>

  <Step title="Configure API Key">
    In StackBlitz, add your API key to the environment:
    
    1. Click the lock icon (ðŸ”’) in the left sidebar
    2. Add `AGENTMAIL_API_KEY` with your key
    3. The app will automatically restart with your credentials
  </Step>

  <Step title="Start Development">
    The development server starts automatically. Edit files and see changes instantly:
    
    ```typescript title="index.ts"
    import { AgentMailClient } from 'agentmail';

    const client = new AgentMailClient({
      apiKey: process.env.AGENTMAIL_API_KEY,
    });

    // Create an inbox
    const inbox = await client.inboxes.create();
    console.log('Inbox created:', inbox.inboxId);
    ```
    
    Changes appear immediately in the preview pane.
  </Step>

  <Step title="Test Webhooks">
    StackBlitz provides a public URL for testing webhooks:
    
    ```typescript title="server.ts"
    import express from 'express';
    import { AgentMailClient } from 'agentmail';

    const app = express();
    app.use(express.json());

    app.post('/webhooks/agentmail', async (req, res) => {
      const { event, data } = req.body;
      console.log('Webhook received:', event, data);
      res.json({ success: true });
    });

    app.listen(3000, () => {
      console.log('Server running at https://[your-project].stackblitz.io');
    });
    ```
  </Step>
</Steps>

## Visual Editors

### Replit

Replit provides a full IDE with visual tools, AI assistance, and built-in deployment.

<Steps>
  <Step title="Create New Repl">
    1. Visit [Replit](https://replit.com)
    2. Click "Create Repl"
    3. Choose "Node.js" template
    4. Name it "agentmail-agent"
  </Step>

  <Step title="Install AgentMail">
    In the Shell tab:
    ```bash
    npm install agentmail
    ```
  </Step>

  <Step title="Connect Integration">
    1. Click the "Integrations" tab in the left sidebar
    2. Search for "AgentMail"
    3. Click "Connect" and paste your API key
    4. AgentMail is now configured!
  </Step>

  <Step title="Use AI Assistant">
    Replit's AI can help you build faster:
    
    Press `Ctrl+K` (or `Cmd+K` on Mac) and ask:
    - "Create a webhook handler for incoming emails"
    - "Add automatic reply functionality"
    - "Implement email sentiment analysis"
    
    The AI will generate code using AgentMail's API.
  </Step>

  <Step title="Visual Debugger">
    Set breakpoints and inspect variables visually:
    
    1. Click line numbers to set breakpoints
    2. Click "Debug" to start debugging
    3. Step through code with visual controls
    4. Inspect variables in the sidebar
  </Step>

  <Step title="Deploy with One Click">
    Click "Deploy" in the top right to make your agent live:
    
    - Automatic HTTPS
    - Environment variable management
    - Custom domains available
    - Auto-scaling included
  </Step>
</Steps>

    ```json title="tsconfig.json"

        "inlineSourceMap": false,
        "inlineSources": true
      }
    }
    ```
  </Step>

### Hot Module Replacement (HMR)

See changes instantly without losing state.

<Steps>
  <Step title="Vite">
    ```typescript title="vite.config.ts"
    import { defineConfig } from 'vite';

    export default defineConfig({
      server: {
        hmr: true,
        watch: {
          usePolling: true,
        },
      },
    });
    ```
  </Step>

  <Step title="Next.js">
    HMR is enabled by default:
    
    ```bash
    npm run dev
    ```
    
    Edit files and see changes instantly at `http://localhost:3000`
  </Step>

  <Step title="Preserve State">
    Use HMR APIs to preserve state during updates:
    
    ```typescript
    if (import.meta.hot) {
      import.meta.hot.accept((newModule) => {
        // Preserve state during hot reload
        console.log('Module updated');
      });
    }
    ```
  </Step>
</Steps>

    Features:

    - Step through test execution
    - View screenshots and traces
    - Debug failures visually
  </Step>

  <Step title="Write Tests">
    ```typescript title="tests/email-agent.spec.ts"
    import { test, expect } from '@playwright/test';
    import { AgentMailClient } from 'agentmail';

    test('should send and receive email', async ({ page }) => {
      const client = new AgentMailClient({
        apiKey: process.env.AGENTMAIL_API_KEY,
      });

      const inbox = await client.inboxes.create();
      
      await client.inboxes.messages.send(inbox.inboxId, {
        to: inbox.inboxId,
        subject: 'Test',
        text: 'Hello',
      });

      // Wait for email to appear in dashboard
      await page.goto('http://localhost:3000/dashboard');
      await expect(page.locator('[data-testid="inbox-count"]')).toHaveText('1');
    });
    ```
  </Step>
</Steps>

## Local Development Dashboards

### Build Your Own Dashboard

Create a visual interface for development and testing.

```typescript title="dashboard.tsx"
import { useState, useEffect } from 'react';
import { AgentMailClient } from 'agentmail';

export function DevDashboard() {
  const [inboxes, setInboxes] = useState([]);
  const [messages, setMessages] = useState([]);
  
  const client = new AgentMailClient({
    apiKey: process.env.AGENTMAIL_API_KEY,
  });

  useEffect(() => {
    loadInboxes();
  }, []);

  async function loadInboxes() {
    const result = await client.inboxes.list();
    setInboxes(result.items);
  }

  async function loadMessages(inboxId: string) {
    const result = await client.inboxes.threads.list(inboxId);
    setMessages(result.items);
  }

  return (
    <div className="grid grid-cols-3 gap-4 p-4">
      {/* Inbox List */}
      <div className="border rounded p-4">
        <h2 className="text-xl font-bold mb-4">Inboxes</h2>
        {inboxes.map((inbox) => (
          <div
            key={inbox.inboxId}
            onClick={() => loadMessages(inbox.inboxId)}
            className="p-2 hover:bg-gray-100 cursor-pointer rounded"
          >
            {inbox.inboxId}
          </div>
        ))}
      </div>

      {/* Message List */}
      <div className="border rounded p-4 col-span-2">
        <h2 className="text-xl font-bold mb-4">Messages</h2>
        {messages.map((thread) => (
          <div key={thread.threadId} className="p-3 border-b">
            <div className="font-semibold">{thread.subject}</div>
            <div className="text-sm text-gray-600">
              {thread.messageCount} messages
            </div>
          </div>
        ))}
      </div>
    </div>
  );
}
```

## Best Practices

<Steps>
  <Step title="Use Environment-Specific Keys">
    Never use production API keys in development. Create separate keys for:
    - Local development
    - Testing/CI
    - Staging
    - Production
  </Step>

  <Step title="Enable Hot Reload">
    Configure your dev server for instant feedback without manual refreshes.
  </Step>

  <Step title="Leverage AI Assistants">
    Use AI tools to:
    - Generate boilerplate code
    - Write tests
    - Debug issues
    - Refactor code
  </Step>

  <Step title="Monitor Network Traffic">
    Keep DevTools open to inspect API requests and responses in real-time.
  </Step>

  <Step title="Use Visual Debugging">

  </Step>
</Steps>
